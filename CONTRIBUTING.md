# 贡献指南

感谢您考虑为Weblog项目做出贡献！以下是一些指导原则，以帮助您为项目贡献代码。

## 开发流程

1. Fork本仓库
2. 克隆您的fork到本地机器
3. 创建一个新分支（`git checkout -b feature/your-feature-name`）
4. 提交您的修改（`git commit -m '添加一些功能'`）
5. 推送到您的分支（`git push origin feature/your-feature-name`）
6. 创建一个新的Pull Request

## 编码规范

- 使用2空格缩进
- 确保代码无错误和警告
- 为新功能编写适当的文档

## Pull Request流程

1. 确保您的PR描述清楚地说明了您的更改
2. 包含任何相关的问题编号
3. 如果您的PR解决了一个已存在的问题，请使用`Fixes #123`语法链接到该问题

## 问题报告

如果您发现了一个问题，请创建一个新的Issue，并包含以下信息：

- 问题的详细描述
- 重现问题的步骤
- 预期行为和实际行为
- 您使用的浏览器和操作系统

再次感谢您的贡献！
